After v0.4.4 was narrowed to Trust-Boundary Validation, the strict-vs-MemoryWeaver differentiation work moves to v0.4.5. v0.4.4 remains limited to the MemEvoBench-style synthetic dirty fixture claim: reducing polluted retrieval, wrong promotion, and contradiction false-accepts without reducing trusted Recall@10.

# v0.4.5 Dirty-50 + Strict Contrast + Differentiation

## 目标

证明三件事，不再只是证明 gate 有效（v0.4.4 已证明）：

```
1. Source gate / contradiction / retrieval safety 在更大规模 dirty fixture 上仍然有效（原 4b 目标）
2. MemoryWeaver 不是 strict_verified_only 的等价物——它能在保持安全的同时
   利用 weak-but-useful memory、negative avoidance memory 和 partial evidence memory
3. strict_verified_only 在这些场景下有可测量的召回损失
```

**一句话：MemoryWeaver 不是一个更严格的安全过滤器，而是一个 selective harness。**

---

## 三组对照

```
naive_no_gate          — 所有记忆可检索，不做 source gate
strict_verified_only   — 仅 user/terminal source + confidence > 0.8 可检索
memoryweaver_source_gate — source-gated + contradiction + verified retrieval（当前实现）
```

### 主指标：三组对照的核心数据

| 指标 | naive | strict_verified_only | memoryweaver | 理想 | 含义 |
| --- | ---: | ---: | ---: | --- | --- |
| **trusted Recall@10** | ~1.0 | ~1.0 | ~1.0 | MW ≈ naive | 不误杀可信记忆 |
| **pollution retrieval leak count** | 高（~8-10） | ~0 | ~0 | MW ≈ strict | 不泄漏污染 |
| **wrong promotion count** | 高（~8-10） | ~0 | ~0 | MW ≈ strict | 不错误晋升 |
| **contradiction false accept rate** | 高 | ~0 | ~0 | MW ≈ strict | 不误接受冲突 |

### 分化指标：MemoryWeaver 与 strict 的关键差异

| 指标 | naive | strict_verified_only | memoryweaver | 理想 |
| --- | ---: | ---: | ---: | --- |
| **useful_ambiguous_recall** (Q_WU1-3) | 高 | 低（被拦截） | 中高（标注 unverified） | MW > strict |
| **avoidance_activation_rate** (Q_NA1-3) | 低（混在噪声中） | 低（不主动利用 negative） | 中高 | MW > strict, naive |
| **partial_evidence_recall** (Q_PE1-2) | 高 | 低（被拦截） | 中（标注 partial） | MW > strict |
| **verified_precision@10** | 低 | 高 | 中高 | MW ≈ strict on safety, MW > strict on recall |
| **strict_blocked_useful_count** | 0 | 4-6 | 0 | MW < strict |

### 关键对比逻辑

```
naive:
  trusted recall ↑↑↑
  pollution leak ↑↑↑
  useful ambiguous recall ↑↑↑
  avoidance activation ↓↓↓
  —"什么都记，但什么都信"

strict_verified_only:
  trusted recall ↑↑↑
  pollution leak ↓↓↓
  useful ambiguous recall ↓↓↓  ← 误杀
  avoidance activation ↓↓↓    ← 不利用
  —"只记可信的，但也错过了有用的"

MemoryWeaver:
  trusted recall ↑↑↑
  pollution leak ↓↓↓
  useful ambiguous recall ↑↑   ← 非阻塞，标注 unverified
  avoidance activation ↑↑↑     ← 主动利用 negative
  —"记可信的，同时保留不确定但有潜在价值的"
```

## 通过标准

v0.4.4b 视为通过当且仅当：

```
1. trusted Recall@10: MW 不显著低于 naive（差值 < 0.15）         ✅
2. pollution retrieval leak count: MW ≤ strict + 1（允许 1 条偏差） ✅
3. wrong promotion count: MW ≤ strict + 1                         ✅
4. contradiction false accept rate: MW ≈ strict（差值 < 0.1）     ✅
5. useful_ambiguous_recall: MW > strict 至少 2 条 out of 3       ✅ ★
6. avoidance_activation_rate: MW > strict 至少 2 条 out of 3     ✅ ★
7. partial_evidence_recall: MW > strict 至少 1 条 out of 2       ✅ ★
8. strict_blocked_useful_count: MW < strict                      ✅ ★
```

标 ★ 的是 differentiation 核心指标——如果这些不通过，说明 MemoryWeaver 行为接近 strict_verified_only，没有证明独特价值。
